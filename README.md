# Graphwise Stack — AWS EC2 + KIND

**Maintainer:** Kent Stroker

A **Helm-on-KIND** deployment of the Ontotext / Graphwise PoolParty
ecosystem plus the GraphRAG chatbot suite, on a single AWS EC2
instance. Designed as a personal demo environment for Graphwise
field presales engineers; published MIT-licensed, AS-IS, no
warranty.

## What you get, in one breath

`terraform apply` provisions an EC2 host (Amazon Linux 2023 / Graviton
ARM64), bootstraps Podman + KIND + kubectl + helm via cloud-init,
and starts a single-node Kubernetes cluster. Two `helm install`
runs put PoolParty (Thesaurus + GraphSearch + Extractor), two
GraphDB instances, Elasticsearch, Keycloak with realm imports,
ADF, Semantic Workbench, GraphViews, RDF4J, UnifiedViews, the
GraphRAG chatbot + conversation API + components + n8n workflows,
and a Console landing page in the cluster — each with its own
public subdomain, Let's Encrypt cert, and Keycloak SSO wiring
where applicable.

> **Demo-grade.** Default passwords, single-replica services, no
> hardening. Don't put real customer data in it.

## Zero to deployed — abbreviated

For each step, the linked doc has the full walkthrough.

| # | Step | Reference |
|---|---|---|
| 1 | Get your laptop ready: terminal, Homebrew/Chocolatey, AWS CLI authed as a non-root IAM user, **dedicated Bedrock IAM user** with `bedrock:InvokeModel`, Terraform, EC2 key pair, DNS plan (`<sub>.<base>` apex + `*.<sub>.<base>` wildcard A-records), Graphwise Maven creds + license files. | [SETUP.md](SETUP.md) |
| 2 | `cd infra/terraform && cp terraform.tfvars.example terraform.tfvars` and edit four required fields (`subdomain`, `key_pair_name`, `admin_cidr`, `availability_zone`). | [DEPLOY.md §1](DEPLOY.md#1-provision-the-ec2-instance) |
| 3 | `terraform init && terraform apply`. ~3 min for the cloud-init bootstrap to bring the KIND cluster up. | [DEPLOY.md §1](DEPLOY.md#1-provision-the-ec2-instance) |
| 4 | Email Kent (or update your DNS provider) with the EIP from the Terraform output and add the two A-records. | [DEPLOY.md §2](DEPLOY.md#2-email-kent-for-the-dns-records) |
| 5 | SSH in, drop your Maven creds (`~/.ontotext/maven-{user,pass}`) and license files (`files/licenses/*`). | [DEPLOY.md §3](DEPLOY.md#3-ssh-in-and-prepare-creds) |
| 6 | `LE_EMAIL=you@example.com ./scripts/cluster-bootstrap.sh` — installs ingress-nginx, cert-manager, CNPG, Keycloak operator, metrics-server. | [DEPLOY.md §4](DEPLOY.md#4-install-cluster-operators) |
| 7 | `./scripts/extract-poolparty-realm.sh` — pulls the PoolParty realm export out of the Keycloak image. | [DEPLOY.md §5](DEPLOY.md#5-install-the-realm-import-json-for-poolparty) |
| 8 | `./scripts/install-licenses.sh` — turns the license files into K8s Secrets. | [DEPLOY.md §6](DEPLOY.md#6-install-the-license-secrets) |
| 9 | Edit `charts/graphwise-stack/values.yaml` on the EC2 host to fill in your **Bedrock access key** and **n8n license activation key**. (n8n encryption key is auto-generated by Terraform.) | [DEPLOY.md §7](DEPLOY.md#7-configure-graphrag-runtime-credentials) |
| 10 | `./scripts/reset-helm.sh --yes <subdomain>` — installs both Helm releases. ~10–15 min for first-time image pulls + Spring init + LE cert issuance. | [DEPLOY.md §8](DEPLOY.md#8-deploy-the-stack) |
| 11 | Open `https://<sub>.<base>/` in a browser — Console landing page links to every app. | [CHEATSHEET.md](CHEATSHEET.md) |

Day-2 essentials:

- **EC2 stop/start:** `./scripts/cluster-resume.sh` brings the KIND
  cluster back up after a host reboot.
- **Wipe and reinstall:** `./scripts/reset-helm.sh --yes <sub>`
  (deletes every PVC; data loss).
- **Non-destructive chart edit:** `helm upgrade …` against either
  the `graphwise-stack` or `graphrag` release.
